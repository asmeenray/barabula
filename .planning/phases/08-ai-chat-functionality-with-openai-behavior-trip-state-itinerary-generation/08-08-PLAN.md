<objective>
Two UX fixes:
1. AI asks ALL missing questions in a single message (bullet list with examples) instead of one per turn — much faster onboarding.
2. Starting a new trip always begins with a completely fresh session — no leftover chat history from previous trips.
</objective>

## Tasks

### Task 1: Update AI prompt — ask all missing questions at once

In `src/lib/ai/prompts/trip-planner.ts`, update `TRIP_PLANNER_RULES`:

Replace the rule:
> "If destination is null, ask for it — this is your only question in this turn."
> "Once destination is known, gather in natural order..."
> "Do not ask more than one question per turn."

With:
```
- In the FIRST turn (when all fields are null), ask destination ONLY — it's the hook.
- Once destination is known but other fields are missing, ask ALL remaining unknown fields in ONE message as a formatted bullet list. Each bullet = one bold question with inline examples. This is the "intake" message.
- After the intake message, if the user answers everything at once, extract all fields and move to ready_for_summary. If they answer partially, acknowledge what you got and ask only what's still missing (one follow-up message, grouped again).
- NEVER ask more than once for a field already in trip_state.
- Example intake message format:
  "Great choice! To build your perfect trip, I just need a few details:
  - **Who's traveling?** Just you, with a partner, a group of friends, or a mix?
  - **When are you going?** Specific dates, or a rough window? *(e.g. 10–15 June, or 'late July for a week')*
  - **Flying from where?** *(London, NYC, Dubai...)*
  - **What's your vibe?** *(History buff, foodie, adventure, beach, nightlife, all of it?)*"
```

Commit: `feat(08-08): ask all missing questions at once in AI intake message`

### Task 2: Clear chat_history on session reset + auto-reset on new trip

The DELETE handler currently only deletes from `trip_sessions`. Also delete `chat_history`:
```ts
// Delete both trip session AND chat history for a clean slate
await Promise.all([
  supabase.from('trip_sessions').delete().eq('user_id', user.id),
  supabase.from('chat_history').delete().eq('user_id', user.id),
])
```

**Update `src/app/(authenticated)/chat/page.tsx`:**

When the page loads with an `initialPrompt` (i.e., user came from landing page hero input — always means a new trip intent), auto-reset the session BEFORE loading history or sending the auto-message:

```ts

useEffect(() => {
  async function initSession() {
    if (initialPrompt) {
      // New trip intent — wipe previous session + history for a clean slate
      await fetch('/api/chat/session', { method: 'DELETE' })
    }

    // Then load history (will be empty after reset)
    fetch('/api/chat/history')
      .then(r => r.json())
      .then(/* existing history merge logic */)
      ...

    // Load trip session state (will be empty after reset)
    fetch('/api/chat/session')
      .then(/* existing session restore logic */)
      ...
  }
  initSession().finally(() => setHistoryLoading(false))

```

Also update the `__reset_session__` sentinel handler to remove the `window.confirm` (replace with a cleaner inline state reset) — since DELETE now clears everything, `window.location.reload()` is the right call to get a clean chat UI. The confirm is fine to keep for safety.
