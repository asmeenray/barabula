In `src/lib/ai/schemas.ts`, update the activity object inside `AIResponseSchema.itinerary.days` to add optional hotel fields. All new fields MUST use `.nullable()` (not `.optional()`) because OpenAI Structured Outputs requires all fields in `required` — `.nullable()` maps to `anyOf: [null, ...]` which is valid. The existing `name`, `time`, `description`, `location` fields stay unchanged.

Add to the activity z.object():
```typescript
activity_type: z.enum(['activity', 'hotel']).nullable(),
hotel_name: z.string().nullable(),
star_rating: z.number().nullable(),
check_in: z.string().nullable(),
check_out: z.string().nullable(),
```

In `src/lib/types.ts`, add `extra_data` to the `Activity` interface (the DB column already exists as JSONB):
```typescript
extra_data: Record<string, unknown> | null
```

Also update `GeneratedItinerary.days[].activities[]` in types.ts to add the same hotel fields as optional (not mandatory for the display type). This interface is used by `FullItineraryPanel` — adding optional fields is backward compatible.

const activities = days.flatMap(day =>
